# V4.8 Industrial Scientific Software Validation Report

Validation time: 2026-05-08 10:31 Asia/Shanghai

Project: `metallocene-epdm-digital-twin`

## 1. Overall Conclusion

Conclusion: **releasable for MVP demonstration and R&D screening use**.

The project passed Python compilation, the full pytest suite, smoke report generation, automated scientific/engineering functional audit, function inventory audit, HTTP entry check and in-app browser UI smoke. No P0/P1 issue was found in this validation pass.

The software remains a research-grade digital twin and process simulator. It is not an industrial design package. Dynamic reactor transfer/heat-transfer coefficients, CFD fields, cubic EOS usage and product-property correlations still require calibration against experimental and pilot data.

- `legacy_archive/`: physically merged legacy process simulator archive.

## 6. Scientific and Mathematical Review

### 6.1 Formula and Equation Checks

Machine-readable equation checks passed through `epdm_sim/equation_tests.py`:

- Arrhenius rate trend: higher temperature does not reduce rate constant.
- ENB pressure factor: pressure increase from 0.7 MPa to 2.0 MPa does not spuriously increase ENB incorporation factor.
- Hydrogen chain transfer: higher H2 does not increase Mw.
- Heat release: higher monomer consumption increases heat duty.
- Wilson K and Rachford-Rice: K values finite/positive and vapor fraction bounded in [0, 1].
- Henry model: saturation concentration increases with pressure.
- Rheology: solids and Mw increase viscosity; higher temperature decreases viscosity.
- Darcy-Weisbach hydraulics: smaller pipe diameter increases pressure drop.
- Fox Tg: finite property result.
- Grade score: closer target gives lower score.

### 6.2 Unit and Dimensional Review

Validated unit-sensitive paths:

- temperature: Celsius input converted to Kelvin where needed
- pressure: MPa/Pa paths checked by preflight/EOS/Henry tests
- mass/mole conversion: template feed `kg/h -> mol/h`
- heat duty: reaction heat reported as positive removal demand
- composition: `C2 + C3 + ENB = 100 wt%` or generic segment composition closes to 100 wt%
- vapor fraction: bounded in [0, 1]
- probability/risk outputs: bounded in [0, 1]

### 6.3 Conservation Review

Passed automated checks:

- total mass balance finite
- component balance finite
- monomer consumed mass close to polymer mass
- product composition closes
- flash mass balance closes
- energy release consistent with consumed moles and polymerization heat

### 6.4 Physical and Chemical Engineering Trends

Passed trend gates:

- H2 increase -> Mw/Mooney not higher
- solids increase -> viscosity higher
- temperature increase -> viscosity lower
- Mw increase -> viscosity higher
- pipe diameter decrease -> pressure drop higher
- flow increase -> pressure drop higher
- conversion increase -> reaction heat higher
- flash pressure decrease -> vapor fraction not lower
- polymer pseudo-component does not enter vapor phase
- ENB feed increase -> product ENB wt% not lower
- pressure increase from 0.7 MPa to 2.0 MPa does not abnormally raise ENB incorporation

Passed:

- illegal temperature, pressure and negative feed are rejected by preflight.
- NaN/inf and negative physical values are checked in functional audit paths.
- dynamic reactor states are projected/fallback-protected.
- Rachford-Rice and EOS paths return bounded/fallback-safe results.
- BDF dynamic mode explicitly falls back in the MVP path to avoid automated audit stalls.

Remaining security hardening:

- add path traversal tests for any future user-provided export/import paths.
- add dependency vulnerability scanning when CI is introduced.

## 11. Performance Review

Passed:

- heavy models are button/task triggered rather than page-load triggered.
- report export reads existing results and writes missing heavy task results as `not_run`.
- BDF stiff solver route is guarded by fallback.
- default CFD grid is lightweight and grid convergence is explicit.

Known bottlenecks:

- full UI E2E and large-file import stress tests are not yet automated.
- posterior, DOE and CFD should stay manually triggered for responsiveness.
- some visualization and equipment helper functions have limited direct unit coverage.
